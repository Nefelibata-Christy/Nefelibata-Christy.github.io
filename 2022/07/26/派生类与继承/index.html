派生类与继承

发表于2022-07-26|更新于2022-07-28
|阅读量:

派生类

继承允许我们依据另一个类来定义一个类，已有的类称为基类/父类，新建的类称为派生类/子类

基类：使用protected成员：可以被本类或派生类的成员使用，在类以外不可以被访问，具有半隐蔽性

派生类：新增的成员使用private

声明

访问控制和继承

派生类可以访问基类中所有的非私有成员。因此基类成员如果不想被派生类的成员函数访问，则应在基类中声明为 private。

访问 public protected private
同一个类 yes yes yes
派生类 yes yes no
外部的类 yes no no

不可以继承

  • 基类的构造函数、析构函数和拷贝构造函数。
  • 基类的重载运算符。
  • 基类的友元函数。

继承类型（通常使用 public 继承）

  • 公有继承（public）：当一个类派生自公有基类时，基类的公有成员也是派生类的公有成员，基类的保护成员也是派生类的保护成员，基类的私有成员不能直接被派生类访问，但是可以通过调用基类的公有和保护成员来访问。
  • 保护继承（protected）： 当一个类派生自保护基类时，基类的公有和保护成员将成为派生类的保护成员。
  • 私有继承（private）：当一个类派生自私有基类时，基类的公有和保护成员将成为派生类的私有成员。

构造函数/析构函数

创建派生类对象时，首先执行基类的构造函数，然后执行派生类的构造函数

撤销派生类对象时，首先执行派生类的析构函数（自动调动基类的析构函数），然后执行基类的析构函数

含有对象成员的派生类的构造函数

含有多个 内嵌对象成员时，调用构造函数顺序又它们在类声明的顺序决定

构造函数执行顺序：
    基类 内嵌对象 派生类
析构函数执行顺序：
    派生类 内嵌对象 基类

虚继承

虚继承与虚基类

虚继承，使得在派生类中只保留一份间接基类的成员。

在继承方式前面加上 virtual 关键字

A：虚基类，不论虚基类在继承体系中出现了多少次，在派生类中都只包含一份虚基类的成员

B,C为A的虚派生 虚派生只影响从指定了虚基类的派生类中进一步派生出来的类，它不会影响派生类本身（影响的是D类）

虚继承的构造函数

最终的派生类 D 来初始化虚基类 A，直接派生类 B 和 C 对 A 的构造函数的调用是无效的。

调用 B 的构造函数时试图将 m_a 初始化为 90，调用 C 的构造函数时试图将 m_a 初始化为 100，

但是输出结果证明了这些都是无效的

构造函数的执行顺序

对于普通继承，按照构造函数出现的顺序依次调用的（自上而下）

而虚基类的构造函数比其他构造函数优先级高，如果同一层次中含有虚基类和非虚基类：

先调用虚基类的构造函数，然后再调用非虚基类的构造函数，最后调用派生类的构造函数

基类和派生类赋值兼容

在公有继承的情况下，有以下三条赋值兼容规则。

  • 派生类的对象可以赋值给基类对象。
  • 派生类对象可以用来初始化基类引用。
  • 派生类的指针可以赋值给基类的指针。

将派生类对象赋值给基类对象，将派生类对象中的基类对象逐个字节地复制到=左边的基类对象中。

在公有继承的情况下，可以说，派生类对象也是基类对象，任何本该出现基类对象的地方，如果出现的是派生类的对象，也没有问题。
